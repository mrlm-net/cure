## Security Scan: PR #141 — OpenAI adapter + SSE parser

**Scan date:** 2026-03-28

### Findings

#### MEDIUM — A10 SSRF / A05 Misconfiguration: HTTP client has no timeout, no redirect limit, unbounded error body

**File:** `internal/agent/openai/openai.go` lines 81, 163–165

**Issue 1 — No timeout:**
```go
httpClient: &http.Client{},
```
`http.Client{}` with no `Timeout` will hang indefinitely if the server stalls and the parent context has no deadline. When `OPENAI_BASE_URL` is overridden (valid for testing), a slow attacker-controlled endpoint blocks the goroutine permanently.

**Issue 2 — Unlimited redirects:**
The default client follows up to 10 redirects. Combined with `OPENAI_BASE_URL` override, this allows SSRF via redirect chain to internal network addresses.

**Issue 3 — Unbounded error body:**
```go
var errBody bytes.Buffer
_, _ = errBody.ReadFrom(resp.Body)
```
No size limit. A malicious server returning a large error body causes unbounded memory allocation.

**Recommended fix:**
```go
httpClient: &http.Client{
    Timeout: 5 * time.Minute,
    CheckRedirect: func(req *http.Request, via []*http.Request) error {
        if len(via) >= 3 {
            return http.ErrUseLastResponse
        }
        return nil
    },
},
```
Replace the error body read:
```go
const maxErrBody = 64 * 1024
var errBody bytes.Buffer
_, _ = io.Copy(&errBody, io.LimitReader(resp.Body, maxErrBody))
```

---

#### LOW — A05: sanitiseError loses error chain

`sanitiseError` wraps with `fmt.Errorf("%s", s)` which breaks `errors.Is`/`errors.As` traversal. The `//nolint:goerr113` suppression is documented, which is acceptable. No immediate fix required — document the tradeoff.

---

#### LOW — A09: SSE scanner 64 KB line cap causes unexpected session termination on oversized lines

**File:** `internal/agent/sseutil/reader.go`

`bufio.NewScanner` defaults to 64 KB max token size. An SSE line exceeding this will return `bufio.ErrTooLong`, terminating the stream with an error. This is fail-safe (no memory exhaustion), but may surprise users. Document the limit in the `Parse` function comment.

---

#### INFO — API key never emitted in events

`openaiAdapter.apiKey` is held only for `sanitiseError` and set in the `Authorization` header. It is not included in any event, log, or error string directly. No action needed.
