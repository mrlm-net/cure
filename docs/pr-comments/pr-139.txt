## Security Scan: PR #139 — pkg/agent tool foundation

**Scan date:** 2026-03-28

### Findings

#### MEDIUM — A02/A09: ToolUseBlock.Input persisted unredacted in session files

**CWE:** CWE-312 (Cleartext Storage of Sensitive Information)

`ToolUseBlock.Input` (`map[string]any`) is fully serialized to JSON session files when `MessageContent` marshals tool-use history. If a tool call is passed credentials or tokens as arguments (e.g. from a skill wrapping an internal API), those values are persisted verbatim under `~/.local/share/cure/sessions/<id>.json` (0600 permissions).

0600 is a good baseline, but does not protect against backup tools that ignore permissions, session files included in bug reports, or multi-user container environments.

**Recommendation:** Add a comment to `ToolUseBlock` documenting that `Input` is persisted and should not carry secret values. Long-term: add a configurable redaction list applied at marshal time.

---

#### LOW — A09: Tool call/result events carry arbitrary string content in event stream

**CWE:** CWE-532 (Insertion of Sensitive Information into Log File)

`ToolCallEvent.InputJSON` and `ToolResultEvent.Result` are emitted as streaming events. When the user passes `--format ndjson`, these are written to log files verbatim. Tool results may contain sensitive API responses or file contents.

**Recommendation:** Document in the event type comments that callers should not assume event content is safe for unredacted persistence.

---

#### INFO — A08 Data Integrity: Session.Tools json:"-" is correct

`Session.Tools []Tool \`json:"-"\`` correctly excludes transient tool state. No action needed.

#### INFO — A03 Injection: MessageContent codec rejects unknown block types with explicit errors

Unknown `type` discriminator values return `fmt.Errorf("agent: unknown content block type %q", t.Type)`. No injection risk. No action needed.
