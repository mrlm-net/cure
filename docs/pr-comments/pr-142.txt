## Security Scan: PR #142 — Gemini adapter

**Scan date:** 2026-03-28

### Findings

#### MEDIUM — A10 SSRF / A02: HTTP client no timeout + no redirect limit + unbounded error body + API key in URL

**File:** `internal/agent/gemini/gemini.go` lines 82, 201–202, 230, 311–312, 331

**Issue 1 — No timeout or redirect limit (same as PR #141):**
```go
client: &http.Client{},
```
Same SSRF-via-redirect risk as OpenAI adapter. `GEMINI_BASE_URL` override + no redirect limit enables redirect chains to internal addresses.

**Issue 2 — API key in URL query parameter:**
```go
url := fmt.Sprintf("%s/v1beta/models/%s:streamGenerateContent?alt=sse&key=%s",
    a.baseURL, a.model, a.apiKey)
```
This is the Gemini API's documented auth mechanism (no header alternative on the public endpoint). However, the key in the URL may be captured by proxies, network monitoring tools, or `GODEBUG=http2debug=2` transport traces. The `sanitiseString` method correctly scrubs the key from error messages. Note for operators: avoid using HTTP inspection proxies when running this adapter.

**Issue 3 — Unbounded error/response body reads:**
Lines 230 and 331 both call `io.ReadAll(resp.Body)` with no size limit:
```go
raw, _ := io.ReadAll(resp.Body)  // line 230 — streamGenerateContent error body
raw, err := io.ReadAll(resp.Body) // line 331 — countTokens response
```

**Recommended fix:**
```go
// HTTP client:
client: &http.Client{
    Timeout: 5 * time.Minute,
    CheckRedirect: func(req *http.Request, via []*http.Request) error {
        if len(via) >= 3 {
            return http.ErrUseLastResponse
        }
        return nil
    },
},

// Error body reads (lines 230, 331):
raw, _ := io.ReadAll(io.LimitReader(resp.Body, 64*1024))
```
